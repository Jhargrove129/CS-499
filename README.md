Professional Self-Assessment
Jason Hargrove
CS 499: Computer Science Capstone
Southern New Hampshire University

Completing the Computer Science program and building my ePortfolio has strengthened my ability to present technical work in a professional, employer-focused way. Across coursework, milestones, and revisions, I learned to translate requirements into working software, evaluate design trade-offs, and communicate results clearly. Developing my ePortfolio pushed me to be intentional about quality organizing my projects, documenting decisions, and demonstrating growth through measurable improvements. This process has helped clarify my professional goals: to contribute as a software engineer who values maintainable design, secure development practices, and continuous learning.
Collaboration and stakeholder communication were recurring themes throughout the program. Even when projects were completed individually, I practiced working as if I were in a team environmentâ€”using clear commit messages, incremental development, and readable documentation that another developer could understand. In discussions and peer feedback activities, I learned to justify technical choices, incorporate critique, and explain complex topics to nontechnical audiences. That same communication skill set is reflected in how I present my ePortfolio: each artifact includes context, the problem being solved, the approach taken, and the impact of enhancements. As a result, my portfolio is not only a showcase of code, but also evidence that I can communicate progress, constraints, and outcomes to stakeholders.
My work with data structures and algorithms developed my ability to think analytically about efficiency, correctness, and scalability. Course activities reinforced the importance of selecting appropriate structures (such as lists, maps, or queues) and designing algorithms that balance performance with readability and maintainability. I also became more comfortable evaluating trade-offs such as time complexity versus implementation complexity and documenting those decisions so they are transparent to reviewers. This mindset supports professional development because it demonstrates that I can deliver solutions that are both technically sound and practical in real-world environments.
Software engineering practices and database awareness were central to my capstone improvements. Through the Weight Tracker application, I focused on code organization, consistent UI behavior, and defensive input handling to improve reliability and user experience. The enhancement process reinforced the value of iterative development: identifying issues through testing and build errors, fixing root causes, and validating changes through repeatable steps. At the same time, I strengthened my understanding of how applications evolve from simple prototypes into maintainable products where modular design, clear naming, and separation of concerns become essential.
Security is an essential professional responsibility, and the program increased my attention to safe defaults and proactive risk reduction. Even in a small application, I practiced input validation, realistic range checks, and avoiding risky shortcuts that can lead to crashes or undefined behavior. More broadly, I learned to approach development with a security mindset: anticipate misuse, protect sensitive data, and design for privacy. This perspective shapes how I will contribute in industry by prioritizing secure coding practices and continuous improvement as requirements and threats evolve.
The artifacts in my ePortfolio fit together as evidence of my growth from implementing features to delivering polished, professional-quality work. The Weight Tracker project demonstrates applied software design and engineering through iterative enhancements, improved stability, and user-centered interface updates. Other supporting work across the program demonstrates my ability to reason about algorithms, communicate technical decisions, and apply best practices consistently. Together, these pieces present a cohesive story: I can design, build, test, document, and refine software solutions while clearly explaining the value they provide.
